# 提示词技巧

## 1. 清晰直接 / Be Clear and Direct

**核心原理**：用明确、具体的语言表达需求，避免模糊指令。

**适用场景**：所有任务（最基础且最广泛有效的技术）。

**为什么好用**：大多数提示失败源于歧义而非模型能力不足。清晰的结构比巧妙的措辞更重要。

**示例**：

```
❌ 不好的提示：
写点关于气候变化的东西。

✅ 好的提示：
写一篇 300 字的科普文章，向初中生解释气候变化的三个主要原因，语言简洁易懂。
```

---

## 2. 多样本提示 / Multishot Prompting

**核心原理**：提供几个输入-输出示例，让模型学习期望的模式和格式。

**适用场景**：特定格式输出、风格模仿、分类任务。

**为什么好用**：模型擅长模式识别，示例比抽象描述更直观有效，无需微调即可快速适应新任务。

**示例**：

```
请将用户评论分类为"正面"、"负面"或"中性"。

示例：
评论：这个产品太棒了，超出预期！
分类：正面

评论：一般般，没什么特别的。
分类：中性

评论：质量太差，用了一天就坏了。
分类：负面

现在请分类：
评论：包装很精美，但功能有点少。
```

## 3. 思维链 / Chain of Thought

**核心原理**：引导模型生成中间推理步骤，将复杂问题分解为顺序执行的小步骤。

**适用场景**：数学计算、逻辑推理、多步问题。

**为什么好用**：模拟人类解题过程；研究表明结构和顺序比内容正确性更重要；通过自然语言组织思维而非单纯增加计算量。

**示例**：

```
❌ 不好的提示：
小明有 15 个苹果，给了小红 3 个，又买了 7 个，吃了 2 个，还剩多少？

✅ 好的提示：
小明有 15 个苹果，给了小红 3 个，又买了 7 个，吃了 2 个，还剩多少？
请一步步思考，列出每一步的计算过程，最后给出答案。
```

## 4. XML 标签结构化 / Use XML Tags

**核心原理**：用 `<tag>` 标签分隔提示的不同部分（指令、示例、数据等）。

**适用场景**：复杂提示、包含多种信息类型的任务。

**为什么好用**：帮助模型理解提示结构，减少信息混淆，便于维护和调试。

**示例**：

```
<instruction>
请根据以下文章内容回答问题，只使用文章中的信息。
</instruction>

<article>
人工智能（AI）是计算机科学的一个分支，致力于创建能够执行通常需要人类智能的任务的系统。机器学习是 AI 的一个子集，它使系统能够从数据中学习并改进。
</article>

<question>
机器学习和人工智能是什么关系？
</question>
```

## 5. 角色设定 / System Prompts

**核心原理**：为模型设定身份、专业背景或行为准则。

**适用场景**：调整语气风格、模拟专家视角、建立对话基调。

**为什么好用**：对语气和风格有帮助，适合风格控制和建立一致的对话基调。

**示例**：

```
你是一位有 20 年经验的儿科医生，擅长用通俗易懂的语言向家长解释医学知识。
请回答时保持耐心、温和，避免使用专业术语，如必须使用请附上解释。

家长问：我家宝宝发烧 38.5 度，需要马上去医院吗？
```

## 6. 预填响应 / Prefill Response

**核心原理**：预先填写模型回答的开头部分，引导输出方向。

**适用场景**：控制输出格式、强制特定开头、JSON/代码生成。

**为什么好用**：直接约束输出起点，减少模型"跑偏"的可能，对格式控制非常有效。

**示例**：

```
用户提示：
请分析这段代码的问题并给出修复建议，以 JSON 格式输出。

预填 Claude 的响应开头：
{
  "问题": "

（Claude 会继续按 JSON 格式完成输出）
```

## 7. 提示链 / Chain Prompts

**核心原理**：将复杂任务拆分为多个独立提示，前一个输出作为后一个输入。

**适用场景**：复杂工作流、多阶段处理、质量要求高的场景。

**为什么好用**：每步可独立优化和调试，降低单次任务复杂度，更易追踪问题来源。

**示例**：

```
任务：将一篇英文论文翻译成中文并写摘要

第一步提示：
请将以下英文段落翻译成中文：[原文]

第二步提示：
请为以下中文内容写一段 100 字的摘要：[第一步的翻译结果]

第三步提示：
请检查以下摘要是否准确概括了原文要点：[第二步的摘要]
```

---

## 8. 长上下文提示 / Long Context Tips

**核心原理**：针对大量输入内容的优化技巧，合理组织和放置长文本。

**适用场景**：处理长文档、多文件分析、大量背景资料。

**为什么好用**：充分利用模型的长上下文能力，正确放置信息可提升检索和理解效果。

**示例**：

```
<documents>
<doc id="1">
[第一份文档内容...]
</doc>
<doc id="2">
[第二份文档内容...]
</doc>
<doc id="3">
[第三份文档内容...]
</doc>
</documents>

请基于以上三份文档，回答以下问题：
1. 三份文档的共同主题是什么？
2. 文档 2 和文档 3 的观点有何不同？

回答时请标注信息来源（如"根据文档 1"）。
```
